# v2 候选功能

> 不是路线图，是**候选池**。每个功能后面有「与 PRODUCT.md 是否冲突」「实现复杂度」「演示价值」三项打分。挑 2-3 个做 v2 即可，**不要全做**。

---

## 评分维度

- **冲突度**：与 PRODUCT.md「不做什么」是否冲突。✅ 一致 / ⚠️ 灰区 / ❌ 冲突
- **实现成本**：1-2 周 / 2-4 周 / 4 周+
- **作品集价值**：⭐(普通) / ⭐⭐(加分) / ⭐⭐⭐(强烈推荐讲)

---

## 🥇 强烈推荐 v2.0 做的

### 1. 价格提醒（本地通知）

**简述**：用户对某币种设阈值，价格突破时本地通知提醒。
- ✅ 与"本地优先"完全一致——通知由本地 WorkManager 触发，不走推送服务
- ✅ 不需要账户、不需要后端
- **冲突度**：✅
- **实现成本**：1-2 周
- **作品集价值**：⭐⭐⭐

**技术点可讲**：
- WorkManager 长周期任务调度
- 通知 channel + 重要级别管理
- Android 13+ POST_NOTIFICATIONS runtime permission
- 用户偏好持久化（DataStore）
- 通知点击 deep link 到详情页（复用现有架构）

**数据模型**：
```kotlin
data class PriceAlert(
    val id: String,
    val coinId: String,
    val direction: AlertDirection,  // ABOVE / BELOW
    val threshold: Double,
    val triggered: Boolean = false,
    val createdAt: Long,
)
```

### 2. 模拟交易（Paper Trading）

**简述**：本地虚拟账户，记录"假装买入/卖出"，计算盈亏。
- ✅ 完全本地，无真实交易
- ✅ 是绝佳的**状态机案例**（订单状态：未成交 / 已成交 / 已平仓）
- **冲突度**：✅（PRODUCT.md "不做交易撮合"指的是真实交易）
- **实现成本**：2-3 周
- **作品集价值**：⭐⭐⭐

**技术点可讲**：
- 业务状态机（你已经有 takeaway #9，可以真实落地）
- BigDecimal 精度计算（takeaway #2 真实应用）
- Room 数据库（v2 才需要，trades 历史数据量大）
- 复杂业务逻辑的可测试性

**核心功能**：
- 初始虚拟资金 $10,000
- 任何币种"市价买入" → 锁定当时价格
- 持仓页：当前价 vs 买入价 = 盈亏%
- "卖出" → 平仓，记入历史
- 历史页：每笔交易、累计胜率、总盈亏
- 重置账户

---

### 3. Wear OS 配套 App

**简述**：手表显示自选币种价格。
- ✅ 与本地优先一致（数据通过 DataLayer 从手机同步）
- 几乎没有真正的 Wear OS crypto App，**差异化拉满**
- **冲突度**：✅
- **实现成本**：3-4 周（学习曲线陡）
- **作品集价值**：⭐⭐⭐（Wear OS 经验在简历上非常吸睛）

**技术点可讲**：
- Wear Compose
- Data Layer API（手机 ↔ 手表同步）
- 长循环 vs 短循环刷新策略（电池）
- Tile + Complication（手表"小组件"）

## 🥈 可以做的（v2.x）

### 4. 法币转换显示

**简述**：默认 USD，可切换 CNY / EUR / JPY / KRW 显示。
- ✅ UI 层换算，无需后端
- 用 CoinGecko 的 `vs_currency` 参数即可
- **冲突度**：✅
- **实现成本**：1 周
- **作品集价值**：⭐⭐

**技术点**：locale-aware 千分位 / 货币符号（你的 PriceFormatter 升级版）

---

### 5. 多看板（Portfolios）

**简述**：除了一个自选，可以建多个分组（"DeFi"、"Layer 1"、"Meme"）。
- ✅ 本地存储扩展
- ⚠️ 增加 UI 复杂度（页面间导航变成树状）
- **冲突度**：✅
- **实现成本**：2 周
- **作品集价值**：⭐⭐

**风险**：增加 UI 复杂度可能违背 PRODUCT.md "极简美学"。慎重。

---

### 6. 颜色盲友好模式

**简述**：把涨绿跌红替换为蓝/橙，或加纹理符号。
- ✅ 纯前端可访问性改进
- **冲突度**：✅
- **实现成本**：3 天
- **作品集价值**：⭐⭐（Accessibility 经验加分）

**实现**：Settings 加 "Color blind mode" 开关，Theme 切换调色板。

---

### 7. 平板 / 折叠屏布局

**简述**：大屏下双栏（左列表 + 右详情），不返回。
- ✅ Material 3 Window Size Classes 适配
- 现代 Android 必备能力
- **冲突度**：✅
- **实现成本**：1-2 周
- **作品集价值**：⭐⭐（**面试官特别爱问**）

**技术点**：`WindowSizeClass`, `SupportingPaneScaffold`, `ListDetailPaneScaffold`

---

### 8. Share watchlist 链接

**简述**：用户可以生成一个 `cryptopulse://watchlist?coins=BTC,ETH,SOL` 链接分享给朋友。
- ✅ deep link 扩展
- ✅ 不收集任何数据，仅 URL
- **冲突度**：✅
- **实现成本**：3 天
- **作品集价值**：⭐

## 🥉 灰区或不建议做的

### ⚠️ 9. 持仓估值（不绑定交易所）

**简述**：手动输入"我有 0.5 BTC"，App 计算总美金价值。
- ⚠️ 接近"账户系统"边缘——但不需要登录
- 用户可能觉得"应该跟踪我交易所的真实持仓"，产品定位会被拉偏
- **建议**：与"模拟交易"二选一，不要都做

---

### ⚠️ 10. 新闻 / 资讯流

**简述**：聚合加密相关新闻、推文。
- ❌ 直接违背 PRODUCT.md "不做内容平台"
- ❌ 需要 server-side aggregation
- **建议**：不做

---

### ❌ 11. 真实交易（连交易所 API）

**简述**：用户填 Binance API key，App 内下单。
- ❌ 涉及 API key 安全，巨大法律 / 责任风险
- ❌ 完全违背 PRODUCT.md
- **建议**：不做

---

### ❌ 12. 注册账户 + 云同步

**简述**：登录后自选跨设备同步。
- ❌ 违背"本地优先"核心理念
- ❌ 引入后端运维成本
- **建议**：不做

## 我的建议

**v2.0 做组合 A 的前两个**：

1. 价格提醒（v2.0）
2. 模拟交易（v2.1）

加起来 4-6 周，**深度足够撑下半年简历**，技术点都是面试常考的。

Wear OS 留到 v3，等 v1 上架积累 100+ 用户后再考虑——届时你已经有 Play Store 数据可以讲，Wear OS 这种"重投入"才值得。
